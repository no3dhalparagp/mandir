WORKFLOW 1: VOUCHER POSTING
─────────────────────────────

User Creates Voucher
  ├─ Select Type: Cash Entry, Payment, Collection, Journal
  ├─ Fill Details: Amount, Accounts, Date
  ├─ Map GL Accounts: Debit & Credit
  └─ Status: DRAFT
    │
    ▼
Validation Layer
  ├─ Client: Zod Schema Check
  ├─ Server: Auth, Business Rules
  └─ Double-Entry: Debits = Credits
    │
    ▼
Save to Database (Draft)
  └─ Voucher Number: VCH-YYYY-MM-001
    │
    ▼
User Reviews & Posts
  └─ Click "Post Voucher"
    │
    ▼
GL Posting Transaction
  ├─ Create Journal Entries
  ├─ Update Account Balances
  ├─ Update Party Ledger
  ├─ Update Bank Accounts
  └─ Record Audit Log
    │
    ▼
Status: POSTED
  ├─ Visible in GL Report
  ├─ Visible in Party Ledger
  ├─ Visible in Cash Book
  └─ Final & Non-Editable

WORKFLOW 4: PUJA REQUEST PROCESSING
─────────────────────────────────────

Devotee Submits Request
  ├─ Puja Type: 7 Types
  │  ├─ Daily, Weekly, Festival
  │  ├─ Custom, Marriage
  │  ├─ Death Ritual, Yagna
  ├─ Deity Name (Optional)
  ├─ Requested Date
  ├─ Number of People
  └─ Special Requests
    │
    ▼
Generate Request Number
  └─ PUJ-00001, PUJ-00002
    │
    ▼
Status: PENDING
  ├─ Awaiting Admin Confirmation
  ├─ Can be Modified by Requester
  └─ Displayed to Admin
    │
    ▼
Admin Review & Confirmation
  ├─ View Request
  ├─ Assign Priest
  ├─ Set Confirmed Date
  ├─ Estimate Cost
  └─ Status: CONFIRMED
    │
    ▼
Puja Execution
  ├─ Priest Performs Puja
  ├─ Duration Recorded
  ├─ Special Notes Added
  └─ Completed
    │
    ▼
Record Actual Cost
  ├─ If Actual < Estimated: Refund
  ├─ If Actual > Estimated: Charge Extra
  └─ If Equal: No Adjustment
    │
    ▼
Status: COMPLETED
  ├─ Receipt Generated
  ├─ GL Posted (if cost)
  └─ Final Record
    │
    ▼
Appear in Reports
  ├─ Devotee Profile
  ├─ Puja Statistics
  ├─ Revenue Reports
  └─ Priest Metrics

WORKFLOW 5: FINANCIAL RECONCILIATION
──────────────────────────────────────

Daily Close
  └─ All Vouchers Posted
    │
    ▼
GL Reconciliation
  ├─ Extract All Journal Entries
  ├─ Sum All Debits
  ├─ Sum All Credits
  ├─ Compare: Debit = Credit?
  └─ If No: Investigate & Fix
    │
    ▼
Trial Balance
  ├─ All Accounts Balanced
  ├─ Assets = Liabilities + Equity
  └─ Verified ✓
    │
    ▼
Generate Reports
  ├─ GL Report
  ├─ Party Ledger
  ├─ Cash Book
  ├─ Bank Passbook
  └─ Financial Statement
    │
    ▼
Record in Audit Log
  └─ Reconciliation Completed
